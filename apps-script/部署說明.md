# GAS + Google Sheets 後端 — 部署說明

讓多位教練用手機填復盤、館長雲端看後台。資料存在一個 Google 試算表，前端由 GAS Web App 服務。

> 重要：GitHub Pages 版只能使用各裝置自己的瀏覽器資料，電腦與手機不會同步。
> 要跨裝置同步，請把教練與館長都導到本說明部署出的 Apps Script Web App 網址（`.../exec`）。

## 檔案
- `Code.gs` — 後端（doGet 服務頁面、讀寫試算表）
- `Index.html` — 前端（＝本機版 + 背景同步層，UI 完全一樣）
- `appsscript.json` — manifest（時區、Web App 權限、V8）

---

## 部署步驟（約 5 分鐘）

### 方式 A：綁定試算表（最簡單，推薦）
1. 開一個新的 Google 試算表，命名例如「跆拳道教練復盤資料」。
2. 上方選單 **擴充功能 → Apps Script**，開啟編輯器。
3. 把 `Code.gs` 內容貼進預設的 `程式碼.gs`。`SS_ID` 保持空字串 `''`（自動用這個綁定的試算表）。
4. 左側「檔案 ➕ → HTML」，命名 **`Index`**（不要副檔名），把 `Index.html` 全部貼進去。
5. 編輯器右上 **部署 → 新增部署作業 → 類型選「網頁應用程式」**：
   - 執行身分：**我**
   - 誰可以存取：**所有人**（教練不用登入 Google）
6. 按「部署」，授權，複製產生的網址（`.../exec` 結尾）。
7. 把網址用 LINE 傳給教練、加到手機主畫面，即可使用。

> 三個分頁 `Reviews` / `Coaches` / `Classes` 會在第一次開啟時**自動建立**並帶入預設教練/班別，不用手動建表。

### 方式 B：獨立腳本 + 指定試算表
1. 先手動開一個試算表，從網址複製 ID（`/d/` 與 `/edit` 之間那串）。
2. 到 [script.google.com](https://script.google.com) 新增專案，貼上 `Code.gs`，把 `SS_ID` 改成你的 ID。
3. 其餘同方式 A 第 4 步起。

## 資料表結構（自動建立）

| 分頁 | 欄位 | 說明 |
|---|---|---|
| **Reviews** | id, date, weekday, coach, role, classType, studentCount, teachAvg, attAvg, hasFlag, needHelp, safety, oneThing, track, createdAt, **json** | 一堂課一列。前面是可在 Sheet 直接篩選/做樞紐的扁平欄，最後 `json` 欄存完整復盤（含每項 KPI 分數、證據、紅旗、AI 結果）。 |
| **Coaches** | name | 教練名單（在系統「設定」頁增刪會同步回這裡） |
| **Classes** | name | 班別名單 |

> 想做更正規化（拆成 teaching_kpi_scores / attitude_flags 等 8 張表）可參考 `../設計說明.md` 第 9 點，但簡易後台用「一列一筆 + json」最好維護。

## 同步邏輯（離線優先）

- 開啟頁面 → 先用本機快取秒開 → 背景 `getBootData()` 抓雲端覆蓋，顯示「☁️ 已連線雲端」。
- 教練按「儲存」→ 先寫本機 → 同時 `saveReviewSrv()` 推到 Sheet（失敗會提示「已存本機」，不會掉資料）。
- 刪除、增刪教練/班別、匯入 JSON 都會自動同步。
- **直接用瀏覽器開本機 `Index.html`（非 GAS 網址）會自動退回純 LocalStorage 模式**，邏輯不變，方便離線測試。

新版排班薪資系統會把完整 DB 存到 `FullDB` 分頁，欄位為 `key / updatedAt / json`。同一個 Apps Script 網址在電腦與手機開啟時，會讀寫同一筆完整 DB，因此可以同步排班、教練復盤、薪資與權限資料。

## 部署常見雷（取自既有筆記）
- HTML 檔名必須叫 **`Index`**（對應 `createHtmlOutputFromFile('Index')`）。
- 改完程式要看到更新，要 **部署 → 管理部署作業 → 編輯（鉛筆）→ 版本選「新版本」** 再部署；只存檔不會更新線上版。
- 手機加到主畫面用的是 `.../exec` 網址，不是 `.../dev`。
- 若整合 LINE 推播，文字裡的 `&` 等字元注意跳脫。
